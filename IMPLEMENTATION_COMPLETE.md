## Key Features

### 1. Semantic Search
- Uses pgvector for cosine similarity search
- 3072-dimensional embeddings
- Configurable threshold (0.65)
- Returns top-3 most relevant chunks
- Fast retrieval (<100ms)

### 2. Question Classification
- Automatically detects if question is about the speech
- Routes to appropriate handler
- Provides helpful redirects for off-topic questions

### 3. Grounded Generation
- LLM only uses retrieved context
- Temperature set to 0.0 for consistency
- Refuses to answer if information not in speech
- Provides accurate, sourced answers

### 4. User Experience
- Clean, intuitive chat interface
- Sample questions for quick start
- Download transcript option
- Real-time responses
- Loading indicators
- Message history

## Deployment Steps

### Step 1: Create Database Table
```bash
supabase migration up
```
Or manually run SQL from `supabase/migrations/20240011_biden_speech_rag.sql`

### Step 2: Ingest Speech
```bash
cd scripts
node ingest-biden-speech.mjs
```

### Step 3: Deploy Edge Function
```bash
supabase functions deploy biden-rag
```

### Step 4: Set Environment Variables
In Supabase Edge Function settings:
- `GROQ_API_KEY`
- `GOOGLE_API_KEY`
- `SUPABASE_URL`
- `SERVICE_ROLE_KEY`

### Step 5: Deploy Frontend
```bash
git add .
git commit -m "Implement RAG for Biden Speech"
git push origin main
```

Then enable GitHub Pages in repository settings.
